AppAcchi Compliant build tools

Use either Maven or Gradle as build tools for your appium test scripts.

Implement testng.xml

Make sure you have defined at least one testng.xml any where in your Appium test project zip. Testng.xml format can be found here

Capabillity parameters as command line arguments

Make sure all the set capability parameters used in your Appium test scripts are to be accepted as command line arguments. i.e., using the java command system.getProperty()

define the desired capability for browser asString broswser = System.getProperty(browser);and then turn into desired capabilities afterwards:

DesiredCapabilities desiredCapabilities = new DesiredCapabilities(); desiredCapability.setBrowserName(browser); mvn -Dbrowser="chrome" test This will help us to control all the desired capabilities

Mandatory desired capabilities to be included

Some of the desired capabilities that should be defined without fail include:

Native ANDROID/IOS App:

  1. UDID mobile_udid - (Ex: mobile_udid=CSDWUJ652JK)
  2. Appium URL appium_url - (Ex: appium_url=http://127.0.0.1:4723/wd/hub)
  3. App File Path App_file_path - (Ex: app_file_path=D:\User\myntra\vegasf\apks\myntra.apk)
  4. Screenshot path screenshot_path - (Ex: screenshot_path=D:\User\myntra\vegasf\screenshots)
  5. Environment environment - (Ex: environment=PROD/TEST/LIVE )
  6. Channel channel - ( Ex: ANDROID/IOS)
  7. Platform version ( device os verison ) platform_version - ( Ex: platform_version=6.0/7.1.1, etc.. )
  8. Device name device_name - ( Ex: device_name=Google Nexus 5s )

ANDROID / IOS Mobile Web:

  1. UDID mobile_udid - ( Ex: mobile_udid=CSDWUJ652JK )
  2. Appium URL appium_url - ( Ex: appium_url=http://127.0.0.1:4723/wd/hub )
  3. Screenshot path screenshot_path - ( Ex: screenshot_path=D:\User\myntra\vegasf\screenshots )
  4. Browser Name browser - ( Ex: browser=Chrome )
  5. Channel channel - ( Ex: ANDROID_MWEB/IOS_MWEB)
  6. App Browser Url app_browser_url - ( Ex: app_browser_url=http://www.myntra.com )
  7. Environment environment - (Ex: environment=PROD/TEST/LIVE )

Desktop Web

  1. App Url app_browser_url - (Ex: app_url=https://www.myntra.com)
  2. Screenshot Path screenshot_path - ( Ex: screenshot_path=D:\User\myntra\vegasf\screenshots )
  3. Browser Resolution browser_resolution_width - ( Ex: browser_resolution_width=1024 ) browser_resolution_height - ( Ex: browser_resolution_height=500 )
  4. Browser Version browser_version - ( Ex: browser_version=64.1 )
  5. Browser Binary File path browser_bin_path - ( Ex: browser_bin_path=D:\User\myntra\browsers\versions\chrome64.1.exe or /Applications/Chrome.app/MacOS/Contents/Google Chrome )
  6. Browser Web Driver Path browser_driver_path - ( Ex: browser_driver_path=D:\User\myntra\browsers\drivers\geckodriver.exe or /Applications/Chrome.app/drivers/geckodriver )
  7. Channel channel - ( Ex: DESKTOP_WEB)
  8. Environment environment - (Ex: environment=PROD/TEST/LIVE )

Capability setting for installing app on device

For setting the capability to install app on a device, please use the below method instead of appPackage

//name of the apk File app = new File(System.getProperty(“app_file_path”));

//the absolute local path to the APK capabilities.setCapability("app", app.getAbsolutePath());

Implement assertions at the test level

suggestion is to Implement Assertions in the test level instead of page level with proper log messages. This will enable you to have rich test centric logs to debug in case of failure analysis.
